Plantilla Celia (diseño con QR, barcode y N° en cuadros de texto)
-----------------------------------------------------------------
  - Abajo / lateral: QR (izquierda) y N° 000000004 en recuadro (derecha).
    El sistema reemplaza el número según el código de contrato CT-###### → 00000000N.
  - Pie de página: código de barras (imagen ancha).
  - QR y barcode se generan con: código de cliente + código de venta (ej. HAP00001-VT-000003).
  - Datos del cliente: se sustituyen los textos de muestra (nombre, DNI, celular, dirección,
    distrito, provincia, departamento, monto total, fecha de firma).

Plantilla base (marcadores Polariss, opcional)
----------------------------------------------
  [[POLARISS_BARCODE]]  → Código de barras + texto CT
  [[POLARISS_WEB]]      → URL empresa
  [[POLARISS_QR]]       → QR + leyenda empresa

Compatibilidad: [[FACTORA_*]] (mismo comportamiento).

Para cambiar el diseño, edite el .docx en esta carpeta y conserve las frases ancla del cuerpo.

Firma y sello fijos
-------------------
El bloque izquierdo de firma, sello y logo de TERRAFIRMA debe permanecer siempre como
imagen fija dentro de la plantilla Word. No colocar marcadores sobre esa imagen ni usarla
para QR/código de barras. El sistema solo debe completar los datos del cliente/predio/venta
y mantener esa firma/sello en el lado izquierdo.

Cronograma de pagos:
  Plantilla: core/pos/templates/contracts/Cronograma.docx
  Use los mismos marcadores de Cliente, Venta, Empresa y Predio.
  Para indicar dónde insertar la tabla de cuotas, escriba:
  {{CRONOGRAMA_TABLA}}

  Si no coloca {{CRONOGRAMA_TABLA}}, el sistema agregará la tabla al final del documento.

  El QR y el código de barras del cronograma se generan igual que en la contrata:
  código de cliente + código de venta. Si el modelo viene de la contrata, conserve
  las imágenes base del QR/barra para que el sistema las reemplace automáticamente.
